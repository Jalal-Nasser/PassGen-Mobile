# iOS App Store Build Isolation

## Build Boundary

- Desktop/web renderer output: `dist`
- iOS Capacitor web fallback output: `dist-ios`
- Capacitor iOS `webDir`: `dist-ios`
- iOS fallback entry: `index.ios.html`
- iOS fallback React root: `src/platform/ios/main.tsx`
- Capacitor entry generated by build: `dist-ios/index.html`

The iOS app uses native SwiftUI for the real UI. The `dist-ios` bundle is only a safe Capacitor fallback and must not import the desktop React app.

## Platform Folders

- `src/platform/ios`: App Store-safe React fallback and iOS-only upgrade surface.
- `src/platform/desktop`: desktop/web payment and activation surfaces.
- `src/platform/shared`: platform detection and shared types.
- `desktop/public`: admin/payment dashboard assets kept outside Vite public copy.

## Appflow Requirements

Appflow can keep using `npm run build`. That script now builds:

1. `dist` for desktop/web.
2. `dist-ios` for Capacitor iOS.
3. `validate:ios-web` against `dist-ios`.

`npx cap sync ios` will copy `dist-ios` because `capacitor.config.ts` sets `webDir: 'dist-ios'`.

## Validation

Run before submitting:

```bash
npm run build
npx cap copy ios
npm run validate:ios-artifact
```

`validate:ios-artifact` checks `dist-ios` and `ios/App/App/public` for desktop payment, license, activation, and dashboard strings.

## Pre-Submit Checklist

- `npm run validate:ios-artifact` passes.
- `ios/App/App/capacitor.config.json` says `webDir: "dist-ios"`.
- `ios/App/App/public` contains the iOS fallback only.
- App Store upgrade UI is native SwiftUI/RevenueCat only.
- No admin dashboard files exist under `public` or `ios/App/App/public`.
- No desktop payment assets exist under `public` or `ios/App/App/public`.
